## 2024-05-18 - [Missing noopener/noreferrer on window.open]
**Vulnerability:** Found a missing `noopener,noreferrer` argument when using `window.open('/privacy.html', '_blank')` in `src/ui/Taskbar.tsx`, while other places in the codebase correctly include it. This exposes the application to potential tabnabbing vulnerabilities, where the newly opened tab can gain access to the `window.opener` object and maliciously redirect the original page.
**Learning:** Security practices must be applied consistently across the codebase. Even if it's an internal route (`/privacy.html`), it's a best practice to always include `noopener,noreferrer` for `_blank` targets to maintain defense-in-depth and avoid regression if the URL later changes to external.
**Prevention:** Enforce consistent use of `noopener,noreferrer` for all `window.open` calls with `_blank` target, perhaps through ESLint rules (like `react/jsx-no-target-blank` for anchor tags, and custom rules for `window.open`).
